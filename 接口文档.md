## 家校通后端 API 接口文档

**Base URL**：`http://192.168.0.101:3001/api`

**统一返回格式**：
```json
{
  "code": 200,
  "message": "提示信息",
  "data": { ... }
}
```
| code | 含义 |
|------|------|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录 / 账号密码错误 |
| 403 | 账号未审核 |
| 404 | 资源不存在 |
| 409 | 冲突（如手机号已注册） |
| 500 | 服务端错误 |

**鉴权方式**：除 `/api/login` 和 `/api/register` 外，所有接口需在 Header 中携带：
```
Authorization: Bearer <token>
```
token 由登录接口返回，有效期 7 天。

### 一、登录

**`POST /api/login`**（无需 Token）

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| account | string | 是 | 手机号 / 学号 / 工号 |

```json

  "code": 200,

    "user": {
      "id": 1,

```

### 二、注册

**`POST /api/register`**（无需 Token）

**公共参数（所有角色必填）**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| phone | string | 是 | 手机号，11 位 |
| password | string | 是 | 密码，至少 6 位 |
| role | string | 是 | `student` / `teacher` / `parent` / `leader` |
| real_name | string | 是 | 真实姓名 |

**按角色额外参数**：

| 角色 role | 额外参数 | 类型 | 说明 |
|-----------|---------|------|------|
| `student` | student_no | string | 学号 |
| | class_id | int | 班级 ID |
| `teacher` | teacher_no | string | 教师工号 |
| | subject | string | 任教科目（如"语文"） |
| `parent` | student_id | int | 绑定的学生 ID |
| | relation | string | 关系（father / mother / grandfather / grandmother / other） |
| `leader` | leader_no | string | 领导工号 |
| | title | string | 职位（如"校长"、"教务主任"） |

**返回**：
```json
{ "code": 200, "message": "注册申请已提交，请等待班主任审核" }
```

> 注册后 `status=0`（待审核），需班主任/领导审核通过（`status=1`）后才能登录。

#### 6.3 请求封装 — `api.ets`

```typescript
// api.ets —— 统一请求层
import http from '@ohos.net.http';
import preferences from '@ohos.data.preferences';
import { contextConstant } from '@kit.AbilityKit';

const BASE_URL = 'http://192.168.0.101:3001/api';

// ---------- Token 读写 ----------

async function getPreferences() {
  return preferences.getPreferences(getContext(), 'jiaxiaotong_prefs');
}

async function saveToken(token: string): Promise<void> {
  const prefs = await getPreferences();
  await prefs.put('token', token);
  await prefs.flush();
}

async function getStoredToken(): Promise<string> {
  const prefs = await getPreferences();
  return await prefs.get('token', '') as string;
}

async function clearToken(): Promise<void> {
  const prefs = await getPreferences();
  await prefs.delete('token');
  await prefs.flush();
}

// ---------- 通用请求 ----------

async function request<T>(
  method: http.RequestMethod,
  path: string,
  data?: object,
  needAuth: boolean = true
): Promise<ApiResponse<T>> {
  const client = http.createHttp();
  const url = BASE_URL + path;

  const header: Record<string, string> = {
    'Content-Type': 'application/json'
  };

  if (needAuth) {
    const token = await getStoredToken();
    if (token) {
      header['Authorization'] = `Bearer ${token}`;
    }
  }

  const options: http.HttpRequestOptions = {
    method: method,
    header: header,
    connectTimeout: 10000,
    readTimeout: 10000,
  };

  // POST / PUT 时把 data 序列化到 extraData
  if (method === http.RequestMethod.POST || method === http.RequestMethod.PUT) {
    if (data) {
      options.extraData = JSON.stringify(data);
    }
  }

  try {
    const resp = await client.request(url, options);
    const result: ApiResponse<T> = JSON.parse(resp.result as string);

    // 401 自动退登
    if (result.code === 401 || result.code === 403) {
      await clearToken();
      // 跳转到登录页，具体路由根据项目自行实现
    }

    return result;
  } finally {
    client.destroy();
  }
}

// GET 请求时把参数拼到 query string
function buildQuery(path: string, params?: Record<string, string | number>): string {
  if (!params) return path;
  const entries = Object.entries(params).filter(([_, v]) => v !== undefined && v !== null && v !== '');
  if (entries.length === 0) return path;
  const qs = entries.map(([k, v]) => `${encodeURIComponent(k)}=${encodeURIComponent(String(v))}`).join('&');
  return path + '?' + qs;
}
```

#### 6.4 页面中使用示例

```typescript
// LoginPage.ets —— 登录示例
import { api, saveToken } from '../api/api';
import { UserData } from '../api/types';

@Entry
@Component
struct LoginPage {
  @State account: string = '';
  @State password: string = '';

  async doLogin() {
    const res = await api.login(this.account, this.password);
    if (res.code === 200) {
      await saveToken(res.data.token);
      // 根据 res.data.user.role 跳转不同首页
      router.pushUrl({ url: 'pages/MainPage' });
    } else {
      AlertDialog.show({ message: res.message });
    }
  }
}
```

#### 6.5 关键注意事项

1. **网络权限**：在 `module.json5` 中申请 `ohos.permission.INTERNET`
2. **Base URL 集中管理**：上线时修改 `api.ets` 中的 `BASE_URL` 常量即可
3. **Token 过期处理**：`request()` 方法中已内置 401/403 自动退登逻辑，按需调整跳转路由
4. **真机调试**：确保手机和服务器在同一局域网，Base URL 中的 IP 是服务端内网 IP
5. **GET vs POST 参数位置**：所有 GET 接口参数拼在 query string，POST/PUT 参数放 body（JSON）
